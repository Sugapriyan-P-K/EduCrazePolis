IP Address

An IP address, or Internet Protocol address, is a numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication. IP addresses serve two primary purposes.

  1. Host or Network Interface Identification:

    • An IP address uniquely identifies a device on a network. This allows for precise communication between devices. Devices can include computers, servers, routers, smartphones, IoT devices, etc.
    • It consists of two parts: the network identifier and the host identifier. The combination of these two parts uniquely identifies a device on a network.

  2. Location Addressing:

    • IP addresses are used to route data packets from the source to the destination across networks. Each router along the way uses the destination IP address to make decisions on where to forward the packet next.
    • IP addresses help in the efficient routing of data across the internet, ensuring it reaches the correct destination.

IPv4 and IPv6:

  • IPv4 (Internet Protocol version 4):

    • Consists of a 32-bit address, represented as four sets of decimal numbers separated by dots (e.g., 192.168.0.1).
    • Due to the increasing number of devices connected to the internet, IPv4 addresses are running out, leading to the development of IPv6.

  • IPv6 (Internet Protocol version 6):

    • Consists of a 128-bit address, represented as eight sets of hexadecimal numbers separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).
    • Designed to overcome the limitations of IPv4, providing an immensely larger address space to accommodate the growing number of devices.

Classes of IP Addresses:

  • Classful Addressing:

    • IP addresses were originally divided into classes (A, B, C, D, E) based on the number of network and host bits.
    • Classes A, B, and C are used for general addressing, while D is reserved for multicast groups, and E is reserved for experimental purposes.

  • Classless Addressing (CIDR):

    • Introduced to improve address space utilization and routing efficiency.
    • CIDR allows for more flexible allocation of IP addresses without strictly adhering to predefined classes.

Private and Public IP Addresses:

  • Private IP Addresses:

    • Reserved for use within private networks and are not routable over the internet.
    • Common private IP address ranges include 10.0.0.0 to 10.255.255.255, 172.16.0.0 to 172.31.255.255, and 192.168.0.0 to 192.168.255.255.

  • Public IP Addresses:

    • Assigned by Internet Service Providers (ISPs) and are routable over the internet.
    • Unique public IP addresses are crucial for devices that need to be directly accessible from the internet.
